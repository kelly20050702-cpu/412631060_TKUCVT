# 虛擬機 (VM) 與 容器 (Container) 比較報告

## 1. VM vs Container核心差異對照表

| 特性 | 虛擬機 (VM) | 容器 (Container) |
| :--- | :--- | :--- |
| **架構層級** | 硬體級別 | 作業系統級別 |
| **作業系統** | 每台 VM 都有完整獨立的 Guest OS | 共用宿主機核心 |
| **啟動速度** | 須經過分鐘級的作業開機流程 | 秒級，僅須啟動一個程序 |
| **磁碟佔用** | 較多 | 較少 |
| **資源損耗** | 較高 ，需預留os消耗的 CPU、RAM 空間 | 極低 ，僅耗用執行時所需的資源 |
| **安全性** | 強度隔離，難受影響 | 略遜 ，共用核心，存在容器漏洞逃逸風險 |

## 2. 本課選擇「VM 裡跑 Docker」的理由

1. **Snapshot功能的威力**：
   當注入故障導致Docker的設定檔系統壞掉時，VM的快照功能快速還原，回歸原本的正常狀態。
   
2. **環境隔離的一致性**：
   若在Windows中跑指令，大家電腦可能有版本差異;若在 Windows 下透過 VMware 建立 Ubuntu 環境，可以確保所有的 Docker 指令都在標準的 Linux Kernel 下執行，避免了不同作業系統間的相容性問題。

## 3. Hypervisor Type 1 vs Type 2 的差異與本課的選擇
本課選擇**Hypervisor Type 2**。雖然Type 1效能較好，但對於學習 Docker 、進行系統破壞還原，它的便利性與安全性更佔居優勢，Type 2允許我們在 Windows 宿主系統下並行操作，且具備強大的snapshot功能。
