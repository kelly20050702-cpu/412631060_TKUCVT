# 虛擬機 (VM) 與 容器 (Container) 比較報告

## 1. VM vs Container核心差異對照表

| 特性 | 虛擬機 (VM) | 容器 (Container) |
| :--- | :--- | :--- |
| **隔離層級** | 硬體級別 (Hardware-level) | 作業系統級別 (OS-level) |
| **作業系統** | 每個 VM 都有獨立的 Guest OS | 共用宿主機的 Kernel |
| **啟動速度** | 分鐘級 (需要完整開機過程) | 秒級 (僅啟動應用程式行程) |
| **資源消耗** | 較高 (需預留 CPU、RAM 空間) | 極低 (僅耗用執行時所需的資源) |
| **安全性** | 強 (完全隔離，適合多租戶) | 中 (共用核心，存在逃逸風險) |

## 2. 本課選擇「VM 裡跑 Docker」的理由

1. **Snapshot功能的威力**：
   當注入故障導致Docker的設定檔系統壞掉時，VM的快照功能快速還原，回歸原本的正常狀態。
   
2. **環境隔離的純淨度**：
   在 Windows 下透過 VMware 建立 Ubuntu 環境，可以確保所有的 Docker 指令都在標準的 Linux Kernel 下執行，避免了不同作業系統間的相容性問題。

3. **靈活性與安全性的平衡**：
   我可以使用 VM 來管理整體的「開發環境」，而使用 Container 來管理具體的「應用程式服務」（如測試 Nginx），這就是現代雲端架構的縮影。

## 3. Hypervisor Type 1 vs Type 2 的差異與本課的選擇
虛擬機提供了一個**安全、可隨時重來的沙盒**；而容器則在沙盒內提供了**極速、標準化的部署能力**。兩者並非競爭關係，而是互補的最佳拍檔。
